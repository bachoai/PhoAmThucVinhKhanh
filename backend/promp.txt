Tôi đang làm dự án khóa luận tên là “Hệ Thống Du Lịch Ẩm Thực Quận 4, TP.HCM” / “Quan4 Culinary Tourism System”.

Hãy giúp tôi hoàn thiện toàn bộ Backend cho dự án này.

## 1. Công nghệ bắt buộc

Backend:

* Ngôn ngữ: C#
* Framework: ASP.NET Core Web API
* Database chính: MongoDB
* MongoDB package: MongoDB.Driver
* Auth: JWT Bearer
* Password hash: BCrypt.Net-Next
* API docs/test: Swagger / OpenAPI
* Upload file giai đoạn đầu: local storage trong wwwroot/uploads
* Cache/rate limit giai đoạn sau: Redis
* Object storage giai đoạn sau: MinIO hoặc S3

Không dùng:

* Không dùng Interface
* Không tạo IService
* Không tạo IRepository
* Không dùng Entity Framework Core
* Không dùng SQL Server
* Không làm frontend
* Không làm mobile trong bước này

Luồng code bắt buộc:
Controller → Service → Repository → MongoDB

## 4. Quy tắc MongoDB entity

Mỗi entity chính tương ứng với một MongoDB collection.

Dùng ObjectId dạng string:

[BsonId]
[BsonRepresentation(BsonType.ObjectId)]
public string Id { get; set; } = string.Empty;

Các quan hệ dùng ObjectId string reference, không dùng navigation property kiểu EF Core.

Ví dụ:

* Poi.CategoryId tham chiếu Category.Id
* Poi.OwnerId tham chiếu User.Id
* PoiLocalization.PoiId tham chiếu Poi.Id
* PoiAudio.PoiId tham chiếu Poi.Id
* OwnerRegistration.UserId tham chiếu User.Id
* OwnerSubmission.OwnerId tham chiếu User.Id
* OwnerSubmission.PoiId tham chiếu Poi.Id nếu submission là update
* AuditLog.UserId tham chiếu User.Id
* AnalyticsEvent.PoiId tham chiếu Poi.Id nếu event liên quan POI

Các object phụ như GeoLocation, OpeningHour, PoiImage, ContactInfo được embed trong Poi hoặc OwnerSubmission.

## 10. Validation

Dùng DataAnnotations trong Request DTO.

Quy tắc:

* Email đúng format
* Password tối thiểu 6 ký tự
* FullName không rỗng
* Name không rỗng
* CategoryId không rỗng khi tạo POI

* Longitude từ -180 đến 180
* PriceRange chỉ nhận "$", "$$", "$$$"

* Limit có giới hạn, ví dụ tối đa 100
* File ảnh chỉ nhận jpg, jpeg, png, webp

* Giới hạn dung lượng ảnh và audio theo appsettings.json
