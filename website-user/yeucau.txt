Tôi đang làm dự án “Quan4 Culinary Tourism System”.

Hiện dự án đã có:

* Backend: `backend/Quan4CulinaryTourism.Api`
* Web Admin: `website-admin`
* Mobile .NET MAUI: `mobile/Quan4CulinaryTourism.Mobile`
* API contract: `docs/API_CONTRACT.md`

Backend đang chạy local tại:
http://localhost:5163

Tôi muốn làm thêm một Public Website cho khách du lịch sử dụng trên trình duyệt, không chỉ trên mobile app.

Hãy tạo project mới tại:

`website`

Không sửa web admin.
Không sửa mobile.
Không sửa backend nếu không cần.
Nếu backend thiếu endpoint nhỏ nào thì báo lại trước khi sửa.

## Mục tiêu

Public Website dành cho khách hàng/du khách, không cần đăng nhập.

Người dùng có thể:

* Xem trang chủ giới thiệu hệ thống
* Xem danh sách địa điểm ẩm thực Quận 4
* Tìm kiếm địa điểm
* Lọc theo category
* Lọc theo priceRange

* Xem bản đồ POI
* Tìm địa điểm gần vị trí hiện tại
* Đổi ngôn ngữ
* Chuyển dark mode/light mode
* Dùng tốt trên desktop và mobile browser
* Có thể cài như PWA nếu làm kịp

## API cần dùng

Đọc kỹ API contract tại:

`docs/API_CONTRACT.md`

Dùng các API public sau:

### Health

GET `/api/health`

### Category

GET `/api/v1/categories`

### POI

GET `/api/v1/poi/load-all?lang=&keyword=&categoryId=&priceRange=`

GET `/api/v1/poi/{id}?lang=`

GET `/api/v1/poi/nearby?lat=&lng=&radius=&limit=&lang=`

GET `/api/v1/poi/search?lang=&keyword=&categoryId=&priceRange=`

### Audio

GET `/api/v1/audio/languages`

GET `/api/v1/poi/{id}/audio?lang=`

GET `/api/v1/audio/pack-manifest`

### Analytics

POST `/api/v1/analytics/collect`

### Maps

GET `/api/v1/maps/pack-manifest`

Lưu ý:

* Public API không cần token.
* Response chung dạng:
  {
  "success": true,
  "message": "...",
  "data": ...
  }
* `GET /api/v1/categories` chỉ trả category active.
* `GET /api/v1/poi/load-all` trả list `PoiResponse[]`, không phải paged response.
* Ảnh/audio backend có thể trả relative URL dạng `/uploads/...`, frontend phải normalize bằng `VITE_API_BASE_URL`.

## Trang NearbyPage

Dùng browser geolocation.

Chức năng:

* Nút “Lấy vị trí hiện tại”
* Chọn radius: 1000, 3000, 5000, 10000
* Gọi API nearby
* Hiển thị danh sách POI gần nhất
* Distance format:

  * < 1000m: “350 m”
  * > = 1000m: “1.2 km”

Nếu người dùng từ chối GPS:

* Hiển thị hướng dẫn
* Có nút dùng vị trí mặc định Quận 4:

  * lat: 10.7578
  * lng: 106.7060

Khi gọi nearby:

* Gửi analytics event `nearby_requested`
* Không gửi raw GPS vào analytics metadata.

## Trang MapPage

Ưu tiên dùng MapLibre GL nếu không cần API key.

Yêu cầu:

* Hiển thị bản đồ
* Load POI
* Marker cho từng POI
* Marker vị trí người dùng nếu có
* Click marker hiển thị popup:

  * name
  * address
  * priceRange
  * button xem chi tiết
* Có filter category/priceRange
* Có nút “Tìm quanh tôi”
* Nếu map library lỗi, fallback sang danh sách POI và không crash.

Nếu dùng MapLibre:

* Dùng style open-source nếu cấu hình được.
* Không hardcode key thật vào source.

## API client

axiosClient:

* baseURL từ `VITE_API_BASE_URL`
* timeout 15000
* response unwrap `ApiResponse<T>`
* error message dễ hiểu

Types:

* ApiResponse<T>
* CategoryResponse
* PoiResponse
* PoiDetailResponse
* NearbyPoiResponse
* PoiAudioResponse
* AudioLanguageResponse
* CollectAnalyticsRequest
* MapPackResponse

## Media URL

Tạo `utils/media.ts`:

* normalizeMediaUrl(url?: string | null)
* Nếu url rỗng → trả placeholder
* Nếu url bắt đầu bằng http → giữ nguyên
* Nếu url bắt đầu bằng `/uploads` → nối với `VITE_API_BASE_URL`
* Nếu url là relative khác → nối base URL phù hợp

Áp dụng cho toàn bộ ảnh/audio.
